## 测试步骤

### 步骤 1: 添加测试账号

1. 点击顶部导航栏的 **"Outlook 邮箱"** 按钮
2. 点击 **"添加邮箱账号"** 按钮
3. 填写测试账号信息：
   - 邮箱前缀: `test`
   - 邮箱域名: `@outlook.com`
   - 密码: `your_password`
   - 备注: `测试授权`
4. 点击 **"添加"**

### 步骤 2: 查看未授权状态

在 Outlook 邮箱列表中，找到刚添加的账号：
- **授权** 列应显示: `未授权` (灰色徽章)
- **操作** 列应显示: `去授权` 按钮 (蓝色)

### 步骤 3: 执行授权

1. 点击 **"去授权"** 按钮
2. 不再弹出独立的 **"Graph API 授权"** 窗口，授权会立即开始
3. 弹窗右侧的 **"授权日志"** 面板：
   - 顶部标题为 **"授权日志"**，右侧状态徽章显示 `授权中`
   - 下方黑色背景区域实时输出授权日志

## 常见问题排查

### 问题 1: 点击 "去授权" 无反应

**检查：**
1. 浏览器控制台是否有 JavaScript 错误
2. 确认按钮的 `onclick` 事件已绑定

**解决：**
```javascript
// 在浏览器控制台执行
console.log(typeof startGraphAuthForAccount);  // 应返回 "function"
```

### 问题 2: 创建任务失败

**可能原因：**
- `account_id` 参数无效
- 数据库中账号不存在
- 邮箱或密码为空

**检查后端日志：**
```bash
# 查看 Flask 控制台输出
```

### 问题 3: SSE 连接中断

**症状：**
日志显示 "授权日志连接中断"

**可能原因：**
- 网络超时
- 服务器进程崩溃
- SSE 连接被防火墙阻断

**解决：**
1. 检查 Flask 控制台的错误日志
2. 确认网络连接正常
3. 重启 Flask 应用

### 问题 4: 授权失败 - Token 换取失败

**常见错误代码：**

- **AADSTS50126**: 密码错误
- **AADSTS50034**: 账号不存在
- **AADSTS50076**: 需要 MFA（多因素认证）
- **AADSTS700016**: 应用未在租户中注册

**解决方法：**
1. 验证账号密码正确性
2. 确认账号类型为个人账号（非企业账号）
3. 关闭账号的 MFA 设置

### 问题 5: 授权流程卡住

**可能卡在的页面：**
- 同意授权页面 (Consent/Update)
- 安全信息添加页面 (proofs/Add)
- 未识别的中间页面

**调试方法：**
1. 查看日志中的 URL
2. 手动访问该 URL，查看页面内容
3. 在 `11_routes_graph_oauth.py` 中添加对应页面处理逻辑

## 性能基准

### 正常授权耗时
- **最快**: 8-12 秒
- **平均**: 12-20 秒
- **最慢**: 20-35 秒

### 影响耗时的因素
1. **网络延迟**: Microsoft 服务器响应时间
2. **中间页面数量**: 同意页面、安全信息页面等
3. **系统代理**: 是否使用代理
4. **服务器性能**: Flask 应用的处理速度

## 安全注意事项

### 日志脱敏
- 前端日志中密码显示为 `******`
- 后端日志使用 `sanitize_error_details()` 处理敏感信息

### Token 存储
- `refresh_token` 在数据库中加密存储
- 使用 `encrypt_data()` 函数加密

### 会话验证
- 所有 API 端点使用 `@login_required` 装饰器
- SSE 流也需要登录会话验证
